Here, the bubble_data is an instance of the CSDModel class. From the data structure, one sees two control variables, labeled as Right Ascension and Declination, and a single one-component uncontrol variable named hlsp_heritage_hst_wfc3-uvis_bubble_nebula_f656n_v1_drc. During the file conversion to the CSD model, we choose to retain the FITS naming convension.

Letâ€™s get the tuples of the controlled and uncontrolled variable objects from the bubble_data instance following,

>>> x = bubble_data.controlled_variables
>>> y = bubble_data.uncontrolled_variables

Because there are two controlled variable objects in x, letâ€™s look at the coordinates of each variable using the coordinates attribute of the respective objects.

>>> x[0].coordinates
[350.31187496 350.31185216 350.31182937 ... 350.04763499 350.0476122 350.0475894 ] deg

>>> x[1].coordinates
[61.12851495 61.12852596 61.12853696 ... 61.25340561 61.25341662 61.25342762] deg

Notice, the descending order of coordinates along x[0] which is a consequence of the reverse attribute set to True for the corresponding Controlled variable object. This is also observed in the data structure view shown above. The component of the uncontrolled variable is accessed through the components attribute.

>>> y00 = y[0].components[0]
