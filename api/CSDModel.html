class csdfpy.CSDModel(filename='', version=None)Â¶

Bases: object

Instantiate a CSDModel class.

The CSDModel class is based on the core scientific dataset (CSD) model. The class is a composition of the DependentVariable and IndependentVariable instances, where every instance of the DependentVariable class is a \(p\)-component independent variable, \(y\) and every instance of IndependentVariable class is a dimension of a \(d\)-dimensional independent variable space \((x_0, x_1, ... x_k, ... x_{d-1})\).

Returns:A CSDModel instance.

add_independent_variable(*arg, **kwargs)Â¶

Adds a new IndependentVariable instance to the CSDModel instance.

There are three ways to add a new independent variable.

From a python dictionary containing valid keywords.

>>> import csdfpy as cp

>>> datamodel = cp.new()
>>> py_dictionary = {
...     'type': 'linearly_sampled',
...     'sampling_interval': '5 G',
...     'number_of_points': 50,
...     'reference_offset': '-10 mT'
... }
>>> datamodel.add_independent_variable(py_dictionary)

From a list of valid keyword arguaments.

>>> datamodel.add_independent_variable(
...     type = 'linearly_sampled',
...     sampling_interval = '5 G',
...     number_of_points = 50,
...     reference_offset = '-10 mT'
... )

From an IndependentVariable instance.

>>> from csdfpy import IndependentVariable
>>> datamodel = cp.new()
>>> var1 = IndependentVariable(type = 'linearly_sampled',
...                            sampling_interval = '5 G',
...                            number_of_points = 50,
...                            reference_offset = '-10 mT')
>>> datamodel.add_independent_variable(var1)
>>> print(datamodel.data_structure)
{
  "CSDM": {
    "version": "0.0.9",
    "independent_variables": [
      {
        "type": "linearly_sampled",
        "number_of_points": 50,
        "sampling_interval": "5.0 G",
        "reference_offset": "-10.0 mT",
        "quantity": "magnetic flux density"
      }
    ],
    "dependent_variables": []
  }
}

For the last method, the instance var1 is added to the datamodel as a reference, i.e., if the instance var1 is destroyed, the datamodel instance will become corrupt. As a recommendation always pass a copy of the IndependentVariable instance to the add_independent_variable() method. We provide the later alternative for it is useful for copying an IndependentVariable instance from one CSDModel instance to another.

add_dependent_variable(*arg, **kwargs)Â¶

Adds a new DependentVariable instance to the CSDModel instance.

There are again three ways to add a new dependent variable instance.

From a python dictionary containing valid keywords.

>>> import numpy as np

>>> datamodel = cp.new()

>>> numpy_array = (100*np.random.rand(3,50)).astype(np.uint8)
>>> py_dictionary = {
...     'components': numpy_array,
...     'name': 'star',
...     'unit': 'W s',
...     'quantity': 'energy',
...     'quantity_type': 'RGB'
... }
>>> datamodel.add_dependent_variable(py_dictionary)

From a lsit of valid keyword arguaments.

>>> datamodel.add_dependent_variable(name='star',
...                                  unit='W s',
...                                  quantity_type='RGB',
...                                  components=numpy_array)

From a DependentVariable instance.

>>> from csdfpy import DependentVariable
>>> var1 = DependentVariable(name='star',
...                          unit='W s',
...                          quantity_type='RGB',
...                          components=numpy_array)
>>> datamodel.add_dependent_variable(var1)

If passing a DependentVariable instance, as a general recommendation, always pass a copy of the DependentVariable instance to the add_dependent_variable() method. We provide the later alternative as it is useful for copying a DependentVariable instance from one CSDModel instance to another.

save(filename, version='0.0.9')Â¶

Serializes the CSDModel instance as a JSON data-exchange data file.

The serialized file is saved with two file extensions. When every instance of the DependentVariable class from the CSDModel instance has an internal subtype, the corresponding CSDModel instance is serialized with a .csdf file extension. If any single DependentVariable instance has an external subtype, the CSDModel instance is serialized with a .csdfe file extension. We use the two different file extensions to alert the end use of the possible deserialization error associated with the .csdfe file extensions when the external data file is inaccessible.

Irrespective of the subtypes from the serialized JSON file, by default, all instances of DependentVariable class are assigned an internal subtype with base64 as the value of the encoding attribute upon the import. The user may, however, change these attribute at any time after the file import and before serializing to a file. The syntax follows,

>>> datamodel.save('myfile.csdf')

where datamodel is an instance of the CSDModel class.
